How it works

A folder in, a brain out.

No pipelines to configure and nothing to label. The graph builds itself from whatever your team already writes.

01

Drop your docs

Markdown, PDFs, Notion or Drive exports — ingest a whole directory in one command.

$ context-graph ingest ./docs
12 files → 214 chunks → embedded

02

The graph builds itself

An LLM extracts entities and relationships. Duplicates merge; every fact keeps its provenance.

Why receipts

A chatbot sounds sure. A ledger shows its work.

Most RAG tools give you a fluent paragraph and a shrug. This one keeps an auditable record of where every claim came from and how often it held up.

Provenance on every claim

Each statement in an answer links back to the exact document and passage it came from. If it can't cite, it doesn't say it.

runbook/deploys.md · chunk 14

Confidence that compounds

A fact seen once is treated differently from a fact confirmed across three docs and a teammate's correction. The graph strengthens with use.

0.62 → 0.68 → 0.74 · 3 observations

A file you own

The whole graph is one local SQLite file. Back it up, diff it, move it, delete it. No vendor database, nothing leaves your network.

.context-graph/graph.db

Teams

One brain for the whole team, two ways to share it.

Pick the lane that matches how your team already works. Both keep the data on infrastructure you control.

Recommended

One shared server

Run a single container; teammates open a link with an access token. Put it on your tailnet and it's reachable from anywhere, exposed to no one.

$ docker compose up -d
$ cat ./data/share-link.txt
# optional: private access from anywhere
$ tailscale serve localhost:4680

No server at all

Git mode

Each person keeps a local graph; a conflict-free merge syncs them through a plain file in your repo. Knowledge travels with your code.

$ context-graph sync
# writes .context-graph/graph.jsonl
$ git pull && context-graph sync
# merges everyone's observations

Quickstart

Running in about a minute.

# with Docker (recommended)
git clone https://github.com/NanoNets/context-graph-engine
cd context-graph-engine
docker compose up -d        # web UI on :4680

# or straight onto your machine
curl -fsSL https://raw.githubusercontent.com/NanoNets/context-graph-engine/main/install.sh | bash
context-graph-web           # then drop docs in the UI

Best extraction quality uses an OpenRouter key (set OPENROUTER_API_KEY). Fully-offline mode works with Ollama — no key, nothing leaves your machine.
